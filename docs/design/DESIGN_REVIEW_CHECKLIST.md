本清单用于向 Oasis 输出审核意见。AI 提供证据、风险、替代方案和实现影响；Oasis 对审美、品牌、构图、范围和视觉版本拥有最终决定权，并负责向 Make 传递最终要求。

缺少固定版本、四屏关键状态或核心流程证据时，审核结论只能是“证据不足”，不得标记为视觉冻结。

## 2. 问题清单格式

| ID | 优先级 | 屏幕/状态/位置 | 当前问题与证据 | 建议修改 | 可接受替代 | 验收方式 | 跨屏/Token/代码/性能影响 | Oasis 决定 |
|---|---|---|---|---|---|---|---|---|
|  | P0/P1/P2 |  |  |  |  |  |  | 接受/修改/拒绝 |

优先级定义：

- **P0**：阻断视觉冻结、驾驶安全、核心流程、权限、共享状态、可访问性或可实现性；
- **P1**：明显损害一致性、可读性、信息层级、交互理解或答辩质量；
- **P2**：不阻断冻结的润色、文案和局部一致性建议。

## 3. 视觉审核

- [ ] 四屏具有同等设计论证和完成质量，但信息密度符合各自职责
- [ ] Cluster 不承载复杂设置、娱乐浏览或长文本
- [ ] HUD 保持最小信息集，不出现菜单、对话框、大图表或持续干扰动画
- [ ] Center 承担复杂任务但不复制整个 Cluster
- [ ] Passenger 不拥有驾驶关键控制，娱乐动效不干扰驾驶者
- [ ] Day/Night 只改变 Token 与材质，不改变业务状态和核心布局
- [ ] 中文长度、数字跳变、空数据、stale 和 offline 不破坏层级
- [ ] 关键状态同时使用图标与文字，不只依赖颜色
- [ ] reduced motion 有明确替代表现

## 4. 三条流程审核

- [ ] 导航由 Center 发起，Cluster 显示路线状态，HUD 显示下一驾驶决策，Passenger 只做允许的旅程协作
- [ ] 告警由 HUD/Cluster 优先，Center 提供详情和允许操作，Passenger 抑制娱乐干扰，解除后统一恢复
- [ ] Passenger 承担媒体与隐私，Center 只显示必要共享状态，娱乐信息不进入 Cluster/HUD
- [ ] 每条流程都有明确起点、中间状态、结束和恢复
- [ ] 降级数据来源和不可用能力不会伪装成实时或已完成

## 5. 技术阻断规则

只有在下列可量化边界被违反时才提出阻断：

- 驾驶安全与端点权限；
- 可访问性和目标距离可读性；
- FastAPI/WebSocket 共享状态单一来源；
- 已冻结协议和核心工作流；
- 性能门槛与目标设备可实现性；
- 资产许可、隐私和离线答辩能力。

每个阻断必须同时提供：

1. 可复现证据或测量方法；
2. 受影响屏幕、流程和实现范围；
3. 至少一个满足硬约束的替代方案；
4. 接受风险时的记录内容与回退方案。

AI 不得以未量化的“最佳实践”替代 Oasis 的审美判断。

## 6. 设计到代码审核

- [ ] 冻结快照可以提取语义 Token，而非依赖散落临时色值
- [ ] 重复元素能够映射为共享组件或受控变体
- [ ] Make 生成代码只作参考，未被假定为生产代码
- [ ] 目标逻辑画布可高保真实现，非目标尺寸有受控适配策略
- [ ] 中文、真实数据、响应式、可访问性或性能偏差均能说明原因
- [ ] 截图回归与人工审核方法明确
- [ ] 实质变更已说明返工成本和回退方案
