# 项目简报

> 状态：早期“城市通勤风险感知”技术基线，并非当前最终课题。当前方向见`../PreDesign/idea-archive/01-当前方向共识.md`。

## 课题名称

主选题：**《面向城市通勤风险感知的智能座舱HMI多模态交互原型设计与实现》**

技术型备选：**《基于计算机视觉与大语言模型的城市通勤智能座舱HMI交互原型设计与实现》**

建议优先使用主选题。它突出应用场景、设计对象和产品形态，不把具体工具误写为创新，也更符合学校“题目准确、具体、体现应用场景”的命名原则。

## 项目定位

面向城市通勤场景的纯软件智能座舱教学验证原型。系统融合车辆模拟信号、本地道路视频中的基础风险信息、驾驶员疲劳与分心状态，通过统一风险事件引擎驱动数字仪表和中控HMI，再由LLM对结构化事件进行解释与行程总结。

系统不连接真实车辆，不进行自动驾驶决策，不用于实际安全控制。

## 目标用户

- 主要用户：城市通勤驾驶者；
- 研究与评测参与者：具备驾驶经验的普通用户或潜在驾驶者；
- 展示对象：指导教师、答辩评委、毕业展观众和潜在用人单位。

## 典型场景

工作日晚高峰，驾驶者在城市道路通勤。前方出现行人，同时驾驶者视线偏移。系统把两个信号转为统一事件，判断为复合高风险，提升HMI警告优先级并给出简短语音解释；行程结束后生成包含事件证据的驾驶报告。

## 核心问题

道路环境、驾驶员状态和座舱交互信息通常分散呈现。传统固定式HMI难以根据复合风险动态调整信息优先级、视觉状态和反馈方式，也难以向用户解释“为什么触发警告”。

## 核心成果

1. 数字仪表与中控HMI软件原型；
2. 车辆状态模拟器与本地视频事件输入；
3. 道路与驾驶员状态统一事件模型；
4. 复合风险判断规则与HMI联动机制；
5. 面向结构化事件的LLM解释、工具调用和行程报告；
6. 源代码、需求/设计/测试/使用说明、演示视频和展板素材。

## 明确不做

- 不连接真实车辆和车载总线；
- 不进行自动驾驶路径规划、控制或安全决策；
- 不自行训练大型视觉模型；
- 不承诺高精度车距或完整交通语义理解；
- 不做复杂3D座舱、导航、娱乐或车辆控制平台；
- 不把YOLO、MediaPipe或LLM API调用本身描述为算法创新。
