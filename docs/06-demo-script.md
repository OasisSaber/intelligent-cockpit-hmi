# 5—8分钟演示脚本

## 演示前准备

- 使用展示电脑提前运行`pnpm test`和`pnpm build`；
- 关闭真实LLM和在线依赖，设置Mock模式；
- 启动`pnpm dev`，打开HMI和API文档；
- 准备录屏视频、静态截图和结构化行程JSON作为三级备份；
- 清理桌面通知，检查字体、音量、电源和网络策略。

## 正常演示流程（约6分30秒）

### 0:00—0:40 课题和边界

说明城市通勤场景、核心问题和教学原型定位。强调不连接真实车辆、不做自动驾驶控制。

### 0:40—1:30 系统总览

展示数字仪表、中控道路画面、驾驶员状态、风险证据、时间线和Mock LLM区域，说明数据链路。

### 1:30—2:20 低风险状态

车辆模拟器持续更新速度、电量、续航和模式；HMI保持低打扰冷色状态。

### 2:20—3:10 单一中风险

Mock事件出现前车中风险或车道偏离，观察HMI切换为中风险，说明规则证据和信息优先级变化。

### 3:10—4:20 复合高风险

等待`125.6s`事件循环出现“前方行人 + 驾驶员注意力偏移”。指出统一事件模型、复合判断、红色高优先级提示和时间线记录。

### 4:20—5:10 行程记录和LLM

点击“生成报告”，展示Mock LLM如何只根据结构化事件解释风险并生成行程摘要。说明真实LLM将在P1阶段接入白名单工具。

### 5:10—6:00 测试与可评价性

展示自动测试结果和评测计划：检测准确率/延迟、风险判断、端到端警告延迟、工具调用与HMI用户测试。

### 6:00—6:30 创新与下一步

总结三个创新点：统一事件与复合风险、风险驱动HMI优先级、结构化事件驱动的LLM解释。下一步接入真实视频算法。

## 离线备用方案

1. 后端无法启动：播放提前录制的完整演示视频；
2. 前端无法启动：展示生产构建截图和API返回JSON；
3. 视觉模型失败：切换Mock事件，不改变风险引擎和HMI链路；
4. LLM不可用：使用内置Mock报告；
5. 摄像头不可用：使用取得授权的座舱内视频；
6. 网络不可用：所有核心资源本地化，二维码只作补充。
