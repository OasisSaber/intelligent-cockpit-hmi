# 项目范围与边界

> 状态：早期风险原型的P0/P1/P2范围。新的多屏HMI范围尚在质询和验证中，见`../PreDesign/idea-archive/`。

## P0 核心模块

1. 数字仪表和中控HMI；
2. 车辆状态模拟器；
3. 本地视频导入接口；
4. 基础车辆、行人和车道检测；
5. 驾驶员疲劳与分心监测；
6. 统一风险事件引擎；
7. 风险等级与HMI实时联动；
8. 结构化行程记录。

## P1 辅助模块

- LLM工具调用和参数校验；
- 基于结构化事件的风险解释；
- 语音输入与播报；
- AI驾驶报告；
- 风险时间线与视频时间点定位。

## P2 可选加分

- 目标跟踪；
- 复合风险规则优化；
- HMI日间/夜间/高风险主题；
- 报告导出PDF；
- 桌面程序封装。

## 明确排除

完整3D汽车、真实车辆接入、自动驾驶控制、高精度距离测量、自训练大型视觉模型、完整交通标志识别、复杂道路语义分割、不可靠情绪识别，以及无关导航/娱乐/车辆控制功能。

## 个人任务与成果边界

当前个人独立成果：需求与交互设计、前后端实现、车辆状态模拟、视频处理接口、统一事件数据结构、风险规则、HMI状态机、LLM工具契约、行程报告、测试评测、技术文档和答辩演示。

预训练YOLO、MediaPipe和LLM属于第三方基础能力。论文中明确其版本、许可、输入输出与本项目集成方式，不声称为自主训练或算法创新。

## 加入跨专业团队后的建议

| 专业 | 可承担内容 | 本人仍独立保留的内容 |
|---|---|---|
| 数字媒体艺术 | HMI视觉规范、动效、屏幕布局 | 前后端系统、事件引擎、状态联动和测试 |
| 视觉传达设计 | 信息图形、展板和视觉识别 | 软件原型和技术文档 |
| 动画 | 数字人形象、表情和提示动效 | LLM工具、数据协议和交互逻辑 |
| 产品设计 | 座舱空间和硬件载体概念 | 纯软件HMI与风险闭环 |

任何组队都要形成共同项目名和每人独立子题，不共享个人报告与评价结果。
