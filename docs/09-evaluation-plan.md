# 测试与评价计划

> 状态：早期风险感知原型的评价清单。多屏HMI需要补充跨屏理解、组件复用、Profile适配和视觉一致性评价。

> 本文件只定义方法和指标。当前没有真实模型实验或用户测试数据，不填写虚构结果。

## 1. 视觉检测

使用合法授权并人工标注的测试视频，按目标和场景分层统计：

- Precision = TP / (TP + FP)；
- Recall = TP / (TP + FN)；
- F1 = 2PR / (P + R)；
- FPS与单帧推理延迟；
- 车辆、行人、车道分别记录漏检和误报；
- 白天/夜间、遮挡、运动模糊等场景单独报告。

不以展示用视频同时调参和报告最终结果；模型版本、分辨率、硬件和阈值必须记录。

## 2. 驾驶员状态

- 闭眼、打哈欠、头部姿态和视线偏移事件的Precision/Recall/F1；
- 状态触发延迟与恢复延迟；
- 不同眼镜、光照和头部角度下的失败案例；
- 只做状态检测，不推断身份或情绪。

## 3. 风险事件引擎

- 场景覆盖率：已有规则覆盖的预定义风险场景比例；
- 风险等级判断正确率：与人工标注等级一致的比例；
- 复合风险识别率；
- 端到端警告延迟：输入事件时间至HMI状态更新；
- 规则冲突、重复事件与短时抖动次数；
- 消融实验：仅道路、仅驾驶员、融合两类状态的结果对比。

## 4. LLM与工具调用

构造固定测试集，每个问题绑定预期工具和参数：

- 工具调用成功率；
- 参数正确率；
- 无效/越权调用率；
- 风险解释是否引用真实事件证据；
- 报告事实一致性：数字、时间、等级与结构化行程记录一致；
- 无密钥、超时和错误响应时的降级成功率。

报告不能增加结构化记录中不存在的事实。人工评价需使用明确量表并保留示例。

## 5. HMI与用户测试

建议任务：识别当前最高风险、解释警告原因、查看事件时间点、完成报告生成。

记录：

- 任务完成率；
- 操作时间；
- 错误次数；
- 警告理解率；
- 主观满意度（统一量表）；
- 开放式用户意见；
- 低/中/高风险状态的识别混淆。

参与人数、招募方式、知情同意和量表需经指导教师确认。未执行前不得填写样本量和结果。

## 6. 软件质量与演示稳定性

- 单元测试：风险规则、数据模型、Mock报告；
- API测试：健康检查、行程记录和报告；
- 前端测试：风险状态和关键交互；
- 连续运行测试：至少覆盖多个Mock循环；
- CPU模式、断网、无LLM密钥和无摄像头降级；
- 构建、启动、中文显示与离线演示检查。
