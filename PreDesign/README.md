# PreDesign 前期设计包

这个目录用于在正式视觉设计和工程实现之前，集中保存方向决策、设计依据、HMI规范草案、可讨论的界面Demo和个人想法。内容服务于导师沟通、设计评审和后续开发，不代表真实车辆安全认证方案。

> 当前项目已经转向“高端汽车多屏HMI母设计系统”。`01—06`与`demo/`主要形成于早期道路风险原型阶段，现作为历史技术基线保留；新的有效方向从`idea-archive/README.md`开始阅读。

## 推荐阅读顺序

1. `idea-archive/README.md`：当前方向、网络案例、组件化框架和工作流；
2. `idea-archive/06-未决事项.md`：后续需求质询和教师确认清单；
3. `01-设计依据与原则.md`：早期公开规范来源和通用设计原则；
4. `02-信息架构.md`：早期风险原型的信息层级；
5. `03-风险状态规范.md`：可复用的风险显示、声音和降噪规则；
6. `04-组件与内容规范.md`：早期组件与中文文案基线；
7. `05-我的Idea池.md`：道路风险与Agent方向的历史Idea；
8. `06-设计评审清单.md`：可继续复用的界面检查表；
9. `demo/index.html`：无需安装依赖的早期交互规范Demo。

## Demo 使用方法

直接打开：

```text
PreDesign/demo/index.html
```

点击顶部“稳定 / 注意 / 高风险”按钮切换风险状态；点击“夜间 / 日间”切换主题；点击“规范标注”查看关键尺寸和层级说明。

也可以在地址后添加参数固定展示状态，例如：

```text
index.html?state=critical&theme=night&spec=1
```

道路画面已接入 Wikimedia Commons 的 CC0 城市行车素材，离线文件位于`demo/assets/urban-dashcam-480p.webm`。视频本身不带目标标注，界面中的车辆框、行人框和风险事件仍是用于验证信息层级的Mock叠加，不代表算法真实检测结果。完整来源、许可与文件哈希见`demo/assets/SOURCE.md`。

目录内的`preview-night-stable.png`和`preview-night-critical.png`是接入CC0视频前的界面基线图，可用于方案对比；后续重新截图时应确保视频已加载到合适画面，并使用新文件名避免覆盖基线。

## 早期Demo设计结论

- 驾驶核心信息只保留速度、当前风险和道路场景；
- 驾驶员状态作为风险证据，不与道路风险分成两个互不相关的警告系统；
- 中风险提示“发生了什么”，高风险提示“发生了什么 + 立即应关注什么”；
- LLM只做结构化事件解释和行程总结，不占据驾驶主界面；
- 正常态使用冷色、低对比装饰；风险色仅用于需要注意的内容；
- 不依赖颜色单独表达风险，始终同时提供文字、图标或位置变化；
- 驾驶中不展示长报告、复杂设置、聊天记录或多层菜单。

## 维护方式

新方向的Idea和决策统一进入`idea-archive/`；道路风险原型的延伸想法仍可保留在`05-我的Idea池.md`。经过本人、导师或用户评审后，再进入正式设计规范和前端代码。
