# 信息架构

## 1. 信息优先级

### L0：任何时刻可见

- 当前车速；
- 当前风险等级；
- 当前最重要的道路/驾驶员事件；
- 系统连接或降级状态。

### L1：一眼可查

- 驾驶模式、电量、续航；
- 驾驶员疲劳/分心摘要；
- 道路目标与车道状态；
- 风险证据摘要。

### L2：需要主动查看

- 风险事件时间线；
- 单个事件详情与视频时间点；
- LLM风险解释；
- 行程报告与改进建议。

### L3：停车态或设置页

- 数据授权与隐私；
- 模型、摄像头和LLM配置；
- 完整日志与诊断；
- 导出、删除和历史报告管理。

## 2. 屏幕分区建议

```text
┌────────────────────────────────────────────────────────────┐
│ 系统状态 / 当前场景                         时间 / Mock标记 │
├───────────────┬──────────────────────────┬─────────────────┤
│ 数字仪表 L0   │ 风险主提示 L0            │ 驾驶员摘要 L1  │
│ 速度          │ 道路画面 / 本地视频       │ 视线、疲劳      │
│ 电量、续航 L1 │ 目标、车道与风险证据      │ 系统边界        │
├───────────────┴───────────────┬──────────┴─────────────────┤
│ 风险事件时间线 L2             │ 通勤助手 / 行程报告 L2    │
└───────────────────────────────┴────────────────────────────┘
```

## 3. 风险态中的布局变化

| 区域 | 低风险 | 中风险 | 高风险 |
|---|---|---|---|
| 风险主提示 | 紧凑状态条 | 提升为醒目卡片 | 固定在视觉中心上方并扩大 |
| 数字仪表 | 完整显示 | 保留速度与核心车辆状态 | 速度保留，装饰信息弱化 |
| 道路画面 | 正常 | 标注单个主要风险 | 仅突出复合风险目标和最短证据 |
| 驾驶员卡片 | 常规摘要 | 风险项高亮 | 合并进复合风险证据 |
| 时间线/助手 | 正常可见 | 降低视觉权重 | 暂时折叠或禁用非必要交互 |

## 4. 核心用户流程

### 行程中

启动Mock/视频 → 显示车辆与感知状态 → 风险事件生成 → HMI状态变化 → 用户快速理解 → 事件写入时间线。

### 追问原因

用户选择事件 → 系统读取结构化事件 → 展示规则、等级和证据 → LLM生成简短解释 → 明确标注“教学验证，不构成真实驾驶建议”。

### 行程结束

事件去重与聚合 → 形成事实摘要 → 生成行程报告 → 用户查看关键事件、时间点和改进建议 → 可选导出。

## 5. 驾驶态与停车态边界

驾驶态允许：查看状态、单次确认提示、语音询问简短原因。

停车态允许：查看长报告、筛选时间线、编辑设置、数据授权、导出和删除记录。
