# 设计评审清单

## 页面目标

- [ ] 页面只有一个主要任务；
- [ ] 驾驶中不需要的信息已移到停车态或行程报告；
- [ ] 用户能在约两秒内说出当前状态和最重要事件；
- [ ] 页面明确标注Mock、视频或真实模型输入模式。

## 信息层级

- [ ] 速度、风险和当前事件属于最高层级；
- [ ] 次要车辆状态不会与风险提示竞争；
- [ ] 高风险时次要内容已压缩或弱化；
- [ ] 详细证据和LLM解释没有塞进主提示。

## 可读性与可访问性

- [ ] 关键文字不小于设计基线；
- [ ] 关键文字与背景对比度目标达到4.5:1；
- [ ] 风险不只依靠颜色表达；
- [ ] 图标配有文字或明确的可访问名称；
- [ ] 支持键盘焦点和`prefers-reduced-motion`；
- [ ] 中文没有乱码、截断或中英文术语混乱。

## 交互

- [ ] 主操作目标足够大且不重叠；
- [ ] 点击后有即时视觉反馈；
- [ ] 加载超过2秒时显示进度或状态；
- [ ] 操作可中断，不强迫用户完成多步流程；
- [ ] 禁用功能说明原因；
- [ ] 高风险时不会弹出复杂对话框或长文本。

## 风险状态

- [ ] 低、中、高三态均有示例；
- [ ] 状态变化不仅是换颜色；
- [ ] 高风险事件包含明确证据；
- [ ] 相同事件已去重；
- [ ] 状态恢复有防抖策略；
- [ ] 没有把模型输出描述成确定事实或真实安全能力。

## LLM与Agent

- [ ] LLM输入来自结构化事件；
- [ ] 工具调用限制在白名单；
- [ ] 回答引用时间点和证据；
- [ ] 无密钥时可降级到Mock；
- [ ] AI生成内容经过人工修改、测试和留痕。

## 交付前

- [ ] 1440×900、1366×768和较窄窗口下无溢出；
- [ ] 日间/夜间主题可读；
- [ ] 核心演示在断网状态可完成；
- [ ] 截图、录屏、Mock数据和启动说明已准备；
- [ ] 所有测试指标来自真实测试，没有预填结论。
