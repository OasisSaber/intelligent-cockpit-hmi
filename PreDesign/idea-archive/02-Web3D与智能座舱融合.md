# Web3D与智能座舱方向融合

## 来源方向A：Web3D与数据可视化

学校方向指南中的“智能座舱数据可视化系统”建议包含：

- 三维汽车模型；
- 实时车辆状态展示；
- 电池管理系统展示；
- 能耗分析；
- 驾驶行为分析；
- Vue3、Three.js、ECharts、MQTT等候选技术。

本项目吸收的是“真实业务场景 + 实时数据 + 三维交互 + 可视化表达”，不把技术清单视为必须全部采用的固定栈。

## 来源方向B：智能座舱与车载交互

学校方向指南给出的相关项目包括：

- 新能源汽车数字仪表系统；
- 智能座舱HMI系统；
- 智能驾驶信息展示系统；
- 车载数据可视化平台；
- AI车载助手、智能语音、多模态交互；
- 疲劳监测、手势控制、氛围座舱与环境控制；
- XR车载娱乐、后排沉浸式体验。

本项目吸收的是HMI、数字仪表、数字座舱和多模态交互方向；AI、手势、XR与疲劳监测保持候选，不在前期全部承诺。

## 融合后的项目公式

```text
多屏HMI设计系统
  + Web3D车辆与空间化状态表达
  + 模拟/实时车辆数据
  + 多车型Profile
  + 跨屏任务流
  + 可选AI与多模态能力
  = 可运行的高端智能座舱交互原型
```

## 各能力的地位

| 能力 | 项目中的角色 | 当前优先级 |
|---|---|---|
| 四屏UI与交互 | 核心设计成果 | 核心 |
| 组件化设计系统 | 支撑多车型一致性与适应性 | 核心 |
| 跨屏状态与任务流 | 证明不是四套静态页面 | 核心 |
| Web3D汽车 | 视觉中心、部件定位与状态入口 | 高价值候选 |
| 实时/模拟车辆数据 | 驱动界面状态与图表 | 核心底座 |
| 电池、能耗、驾驶行为 | 可用于SUV/商务车的业务场景 | 候选 |
| AI车载助手 | 受组件和任务约束的交互层 | 候选 |
| 疲劳、风险感知 | 可复用的早期技术模块 | 历史基线/候选 |
| XR与后排娱乐 | 后续扩展方向 | 实验 |

## 防止项目退化

- 不能只做旋转汽车和换色；
- 不能只做若干图表组成的数据大屏；
- 不能只做四套高保真静态稿；
- 不能把调用Three.js、MQTT或LLM本身描述为创新；
- 必须同时呈现设计系统、实时状态、跨屏逻辑和可验证的车型差异。
