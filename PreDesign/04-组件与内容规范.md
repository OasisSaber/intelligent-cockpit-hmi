# 组件与内容规范

## 1. 设计Token

### 色彩

| Token | 建议值 | 用途 |
|---|---|---|
| `--bg` | `#05070B` | 夜间主背景 |
| `--surface` | `#0C131D` | 常规卡片 |
| `--surface-raised` | `#121D29` | 抬升卡片 |
| `--text` | `#F4F7FB` | 关键文字 |
| `--muted` | `#9AA9BA` | 次要说明 |
| `--accent` | `#58D8D3` | 正常态强调 |
| `--stable` | `#58D3A4` | 稳定状态 |
| `--attention` | `#FFC857` | 中风险 |
| `--critical` | `#FF626D` | 高风险 |

风险色只能用于风险相关组件，不作为品牌色大面积使用。

### 字体

- 中文：系统无衬线字体，优先`Microsoft YaHei UI`、`Noto Sans SC`；
- 数字：允许使用等宽或窄体数字，但要保证`0/8`、`1/7`易区分；
- 风险标题：28—36px，600字重；
- 关键数据：36—64px；
- 正文：20—24px；
- 次要标签：14—18px，不承担关键决策。

### 间距与形状

- 基础间距：8px；
- 卡片内边距：24px；
- 区块间距：20—24px；
- 主卡片圆角：24px；
- 次级卡片圆角：16px；
- 标签圆角：999px；
- 描边：1px，风险状态可提升到2px。

## 2. 核心组件

### 风险主提示 Risk Banner

必须包含：状态词、事件结论、可选动作提示、风险图标。高风险时最多展示两条证据，不显示模型置信度小数。

### 数字仪表 Digital Cluster

速度为视觉锚点；电量、续航、模式为二级信息；风险发生时不隐藏速度，不播放复杂仪表动画。

### 道路画面 Road View

目标框只显示与当前风险有关的对象。Mock、视频、模型推理三种模式必须有明确标识，避免把模拟结果误认为真实检测。

### 驾驶员监测 Driver Monitor

使用“正常 / 偏移 / 疲劳趋势”等状态词，不使用羞辱或归因式文案；不展示人脸身份和不可靠的情绪判断。

### 事件时间线 Event Timeline

默认仅显示最近3—6条非低风险事件；每条包含等级、事件、时间点；点击定位属于P1，未实现时明确标注。

### 通勤助手 Assistant

驾驶中默认折叠，只允许短问题和短回答；长报告放在行程结束后；始终显示Mock/在线提供方和系统边界。

## 3. 图标规则

- 图标必须配文字，除非含义经过反复使用且无歧义；
- 同一语义只使用一种图标；
- 线性图标统一描边宽度；
- 高风险图标不使用装饰性旋转；
- 图标不能成为区分低/中/高风险的唯一方式。

## 4. 动效规则

- 状态切换：150—300ms；
- 常规卡片不持续浮动；
- 高风险允许一次边框脉冲，禁止无限闪烁；
- 支持`prefers-reduced-motion`；
- 数据更新优先使用数字平滑变化，避免整个页面重排。

## 5. 中文内容规则

- 结论前置：先写“前方行人”，再写原因；
- 一个组件只使用一个术语，例如统一使用“注意力偏移”，不混用“分神/走神/分心检测”；
- 避免绝对化：不用“安全”“一定”“完全正常”；
- 明确模拟：使用“Mock事件”“教学验证”“不构成真实驾驶建议”；
- 按钮使用动作词：查看原因、生成报告、定位事件；
- 禁用按钮说明原因，不只改变透明度。
